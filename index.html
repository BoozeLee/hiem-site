7 Core Axioms

01
Correctness before clevernessIf a simpler solution is provably correct, use it.
02
Security is mandatoryEvery new surface area is a threat surface. Treat it that way.
03
Delete fluentlyNo commented-out code. No // TODO: remove later. Remove it now.
04
YAGNI ruthlesslyAbstractions need at least two call-sites today.
05
One PR, one intentRefactor + bug-fix + feature = three PRs, not one.
06
Readable > briefSpell-out names; abbreviations for id, url, ctx only.
07
APIs are contractsChanging a public signature breaks every importer. Version bump first.

Phase Discipline

Before touching any line of code, execute all five phases in order.

Phase 1
Ground truth
Read before writing · Trace call chain · Check tests · History
Phase 2
Design & plan
One-paragraph plan · Todo tracking · Split large changes
Phase 3
Implement
Smallest correct change · Type-check · Tests · Commit
Phase 4
Verify
Full suite · Linter · Diff scan · Threat-model
Phase 5
Cleanup
Remove debug · Unused · Dead code · Update docs

Pre-Close Self-Check

Before marking any task complete, silently verify each box.

  • ☐ Read every affected file, not only the ones edited
  • ☐ Type-check + test + lint in sequence
  • ☐ Baseline test results captured
  • ☐ No stale console.log / dbg! / commented-out code
  • ☐ Public API documentation added or updated
  • ☐ Every security finding in the diff called out
  • ☐ CHANGELOG updated for user-facing changes
  • ☐ Principal-engineer-ready — would sign without amendments

Any unchecked box means go back and fill it before closing.
